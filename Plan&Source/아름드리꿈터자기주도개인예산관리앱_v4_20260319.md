# 🌳 중랑구청 자기주도 개인예산 관리 앱 — 통합 기획·구현 계획서

> **문서 버전**: v4.0 (통합본) | **작성일**: 2026-03-19 | **기준 문서**: 구현 계획서 (2026-03-19) + 바이브코딩 계획서 v3.0 (2026-03-12)

---

## 1. 프로젝트 개요

### 1.1 배경 및 목적

성인 발달장애인 당사자의 개인예산제 운영에서 가장 중요한 과제는 **당사자의 자기결정권을 보장하면서도** 실제 생활 속 예산 관리를 이해하기 쉬운 방식으로 지원하는 것입니다.

기존의 일반 예산 관리 도구는 숫자 중심·표 중심·관리자 중심으로 설계된 경우가 많아, 의사소통에 어려움이 있거나 추상적인 금액 변화 이해가 어려운 당사자에게는 접근성이 낮습니다. 또한 계획서·평가서·영수증·사용 내역·파일 첨부 자료가 분산되어 있어 당사자와 지원자가 함께 확인하기 어렵습니다.

이에 따라 본 프로젝트는 **예산 변화의 시각화, 선택지 비교, 영수증 OCR, 월별 평가, 파일 링크 공유, AI 분석 보조** 기능을 포함한 반응형 웹 앱을 구축합니다.

### 1.2 프로젝트 목적 (5대 목적)

1. **자기주도성 강화** — 당사자가 자신의 예산을 스스로 계획하고 확인하는 주체가 되도록 지원
2. **선택의 가시화** — 오늘의 선택에 따라 예산과 시간이 어떻게 달라지는지 이미지와 쉬운 문장으로 즉시 이해 가능하도록 지원
3. **지원의 체계화** — 지원자가 계획·기록·평가를 작성하고 당사자는 이를 쉬운 정보 형식으로 함께 확인할 수 있도록 지원
4. **자료의 통합 관리** — 예산 사용·영수증·평가·첨부파일·AI 안내를 당사자별로 체계적으로 관리
5. **현장 활용성 확보** — 모바일·태블릿·PC 환경에서 모두 사용할 수 있는 반응형 웹 시스템 구축

### 1.3 핵심 가치 및 원칙

| 핵심 가치 | 내용 |
|-----------|------|
| **자기결정 중심** | 관리받는 구조가 아닌 스스로 선택하는 구조 |
| **지원 중심** | 서비스 제공보다 선택과 이해를 돕는 지원 |
| **직관적 시각화** | 복잡한 그래프보다 이미지와 크기 변화 중심 표현 |
| **성인 존중** | 유아적 표현 지양, 성인 친화적·존중하는 문체와 디자인 적용 |
| **보편적 이해** | 말이 적어도, 숫자가 어려워도 이해 가능한 정보 구조 |
| **현장 적합성** | 중랑구청의 개인예산제 운영 구조와 평가 체계 반영 |

## 2. 기관 운영 구조

중랑구청의 개인예산제는 **연 150만원** 기준으로 운영합니다.

- **운영 기간**: 2026년 3월 ~ 2026년 12월
- **월 예산**: 20만원 (이 중 15만원을 당사자 모임통장 체크카드 예산으로 반영·입급)
- **체크카드 보관**: 당사자 직접 보관 또는 지원자 대리 보관 (당사자 선택)
- **다중 재원 지원**: 재원이 2개인 당사자를 위해 **통합 보기**와 **재원별 보기** 모두 지원

## 3. 사용자 정의 및 역할

### 3.1 역할별 사용자 구분

#### 👤 당사자 (Participant)
성인 발달장애인 이용자.

- 본인 예산 현황 확인 (잔액 시각화)
- 오늘의 선택지에 따른 예산 차이 비교
- 영수증 사진 업로드 (임시 반영)
- 본인 계획·평가의 쉬운 설명 확인
- 본인에게 연결된 파일 링크 열람

**핵심 니즈**: "남은 돈이 얼마나 있는지 눈으로 보고 싶다", "어떤 방법을 고르면 돈이 얼마나 달라지는지 바로 알고 싶다", "어려운 설명 없이 쉽게 이해하고 싶다"

#### 🧑‍💼 지원자 (Supporter)
당사자를 직접 지원하는 사회복지사.

- 당사자 예산 정보 등록·수정 (통장사본 및 통장표지 첨부 포함)
- 계획 등록·수정
- 회계장부(Data Grid) 형태의 당사자별 사용 내역 결산 및 잔액 관리 (CRUD)
- 영수증 OCR 결과 검토·확정 관리
- 카카오뱅크 통장사본, 평가도구 근거자료 모음, 영수증 등 각종 증빙 관리
- PCP 4+1 형식 기반 월별 평가 작성
- AI 분석 결과 확인 및 파일 링크 연동
- 당사자 맞춤 활동·선택지 제안 관리

**핵심 니즈**: "실무용 회계장부처럼 여러 당사자의 예산과 거래를 한 화면에서 관리하고 싶다", "통장사본, 근거자료 등 흩어져 있는 자료를 한곳에서 올리고 관리하고 싶다", "영수증과 평가 기록 입력 시간을 줄이고 싶다"

#### 🔐 관리자 (Administrator)
시스템 및 운영 관리 담당.

- 기관 기본 설정 및 시스템 공통 문구·기준값 관리
- 사용자 계정 및 역할 관리
- 당사자 등록 및 예산 구조 설정
- 전체 당사자 정보 조회 및 데이터 운영 점검

### 3.2 접근성 고려사항

1. 큰 터치 영역 사용
2. 한 화면에 한 가지 핵심 판단 중심 배치
3. 아이콘 + 텍스트 + 이미지 동시 제공
4. 복잡한 표와 그래프 최소화
5. 삭제 및 변경 전 확인 단계 제공
6. 일관된 버튼 위치와 문구 유지
7. 성인 친화적 색상과 일러스트 사용

## 5. 핵심 기능 명세

### 5.1 당사자별 예산 프로필 관리
**목적**: 당사자마다 개별 예산 구조를 분리 저장·운영

- 당사자 등록, 운영 기간 설정, 월·연 예산 설정
- 재원 수 설정 및 재원별 예산 입력
- 통합 예산 자동 계산, 경고 기준 설정
- 담당 지원자 지정, 첨부 문서 링크 등록

### 5.2 홈 대시보드 및 잔액 시각화
**목적**: 현재 예산 상태를 당사자가 한눈에 이해할 수 있도록 표시

**1% 단위 시각화 로직**

```
월 기준: 150,000원 → 100단계 (1단계 = 1,500원)
연 기준: 1,500,000원 → 100단계 (1단계 = 15,000원)
```

**상태 문구 기준**

| 잔액 비율 | 표시 문구 |
|-----------|-----------|
| 81~100% | 예산이 넉넉합니다 |
| 61~80% | 예산이 안정적입니다 |
| 41~60% | 예산을 살펴보며 쓰고 있습니다 |
| 21~40% | 남은 돈이 줄고 있습니다 |
| 11~20% | 남은 돈이 적습니다 |
| 0~10% | 모든 예산을 사용했습니다 |

**남은 날짜 반영 속도 안내 문구 예시**
- 이번 달 남은 날짜에 비해 예산이 여유 있습니다
- 지금 속도로 쓰면 계획에 맞게 사용할 수 있습니다
- 이번 달 남은 날짜를 생각하면 조금 천천히 쓰는 것이 좋습니다

**추가 표시 요소**
- 통합 보기 / 재원별 보기 전환
- 돈주머니·돈뭉치·지갑 이미지 변화 (잔액 비율 연동)
- 오늘 계획 보기 / 영수증 올리기 / 이번 달 평가 보기 버튼

> ✅ **구현 현황**: `budget-visuals.ts` 시각화 로직 완료, `page.tsx` 잔액 카드 UI 완료

### 5.3 오늘 계획 비교 (선택 시뮬레이션)
**목적**: 활동 방법에 따른 비용·시간 차이를 당사자가 직접 비교하고 선택

```
활동명 입력 또는 추천 활동 선택
  → 방법 카드 2~3개 표시
     · 예상 비용 / 예상 시간
     · 선택 후 남는 오늘 예산
     · 선택 후 남는 이번 달 예산 / 연 예산
     · 이미지 차이
     · 쉬운 설명 문구
  → 미리보기 중심 제공 (즉시 차감 안 됨)
  → 지원자가 실제 기록으로 전환 가능
```

**예시: 활동 — 카페 가기**
- 방법 A: 택시 → 비용 높음 / 시간 짧음
- 방법 B: 버스 → 비용 낮음 / 시간 김
- 방법 C: 도보 + 버스 → 비용 낮음 / 시간 더 김

### 5.4 활동 및 사용 내역 관리
**목적**: 지원자와 관리자가 당사자별 사용 내역을 관리

**입력 항목**: 날짜, 활동 내용, 사용 금액, 분류, 메모, 결제 수단, 영수증 이미지, 입력자, 재원 구분, 관련 계획, 관련 평가 메모

**원칙**
- 관리자·지원자만 CRUD 가능
- 당사자는 영수증 제출 및 확인 참여 중심

### 5.5 영수증 OCR
**목적**: 영수증 정보를 자동 추출하여 입력 부담 감소

**OCR 모델**: OpenAI GPT-5 기반 (실제 도입 전 비용 검증 PoC 선행)

**처리 흐름**
```
당사자 또는 지원자 → 영수증 사진 업로드
  → Supabase Storage에 이미지 저장
  → Edge Function → OpenAI GPT-5 OCR
     → 날짜, 금액, 사용처 추출
  → 임시 반영 상태로 저장 (status: pending)
  → 지원자 검토 화면:
     → 결과 확인 / 수정 / 확정
     → 확정 시 status: confirmed → 잔액 차감
```

**운영 원칙**: 당사자 업로드만으로 즉시 확정 차감하지 않음. 반드시 지원자 검토 단계 거침.

### 5.6 달력형 사용 내역
**목적**: 날짜별 흐름 중심으로 예산 사용 내역 이해

- 월간 달력 표시, 사용 내역 있는 날 표시
- 날짜 클릭 → 상세 내역 (임시/확정 상태 구분)
- 하루 총 사용액 표시, 재원별 사용 구분 표시

### 5.7 월별 평가 및 AI 분석
**목적**: 지원자의 평가 기록과 당사자용 쉬운 설명을 연결

**평가 방식**
- 지원자가 PCP(사람중심생각) 4+1 형식 기반 서술형 평가 작성
- 개별 평가도구는 참여자별 별도 붙임 자료로 관리
- 시스템에는 관련 파일 링크 연결

**AI 출력 1: 지원자용 분석**
- 일별·주별·월별·분기별 사용 흐름 분석
- 계획 대비 실제 사용 경향 분석
- 반복 선택 패턴 분석
- 위험 구간 및 편차 구간 안내
- 다음 지원 시 참고 포인트 제안

**AI 출력 2: 당사자용 쉬운 설명**
- 이번 기간에 돈을 어떻게 사용했는지 쉬운 설명
- 어떤 선택이 돈을 더 아끼는 데 도움이 되었는지 설명
- 다음 선택 시 참고할 수 있는 짧은 제안
- 어려운 분석 용어 제외, 간단한 문장 제공

### 5.8 문서 및 파일 링크 첨부
**목적**: Google Drive 내 자료를 당사자별로 연결하여 공유

- 지원자·관리자가 Google Drive 링크 등록
- 원본 링크만 표시 (자동 권한 공유 미포함)
- 당사자 화면에서 허용된 링크만 열람
- 지원 파일 유형: hwp, pdf, docx, jpg, png

**활용 자료 예시**: 계획서, 월별 평가서, 개별 평가도구, 선호 기록, 활동 참고자료

### 5.9 활동 및 선택지 제안
**목적**: 첨부자료와 기존 기록을 바탕으로 새로운 활동·선택지 제안

**제안 기준**
- 당사자의 기존 계획 / 선호 자료 / 과거 활동 이력
- 사용 예산 흐름 / 첨부 문서 내용

**운영 원칙**: 초기에는 지원자 보조 기능으로 제공. 이후 당사자용 쉬운 추천 카드로 확장 가능하게 설계.

## 9. 데이터베이스 설계

### 9.1 테이블 구조

```
PostgreSQL (Supabase)
├── profiles                          ← 사용자 프로필
│   └── id: UUID (PK, auth.users FK)
│       ├── role: TEXT                (admin | supporter | participant)
│       ├── name: TEXT
│       └── created_at: TIMESTAMPTZ
│
├── participants                      ← 당사자 예산 프로필
│   └── id: UUID (PK, profiles FK)
│       ├── monthly_budget_default: NUMERIC  (기본 월 예산, 150,000)
│       ├── yearly_budget_default: NUMERIC   (기본 연 예산, 1,500,000)
│       ├── budget_start_date: DATE
│       ├── budget_end_date: DATE
│       ├── funding_source_count: INTEGER
│       ├── alert_threshold: NUMERIC
│       ├── assigned_supporter_id: UUID      (담당 지원자 FK)
│       ├── bank_book_copy_url: TEXT         (통장사본 Storage URL)
│       ├── bank_cover_url: TEXT             (통장표지 Storage URL)
│       └── created_at: TIMESTAMPTZ
│
├── funding_sources                   ← 재원별 예산
│   └── id: UUID (PK)
│       ├── participant_id: UUID
│       ├── name: TEXT
│       ├── monthly_budget / yearly_budget: NUMERIC
│       ├── current_month_balance / current_year_balance: NUMERIC
│       └── created_at: TIMESTAMPTZ
│
├── transactions                      ← 사용 내역
│   └── id: UUID (PK)
│       ├── participant_id / funding_source_id: UUID
│       ├── date: DATE
│       ├── activity_name / amount / category / memo: TEXT/NUMERIC
│       ├── payment_method: TEXT
│       ├── receipt_image_url: TEXT
│       ├── status: TEXT             (pending | confirmed)
│       ├── creator_id: UUID
│       └── created_at / updated_at: TIMESTAMPTZ
│
├── plans                             ← 계획 (미구현)
│   └── options: JSONB               ({method, cost, time, description}[])
│
├── evaluations                       ← 월별 평가 (미구현)
│   └── supporter_note / easy_summary: TEXT
│       ai_analysis: JSONB
│
├── file_links                        ← 파일 링크 (미구현)
│   └── url: TEXT (Google Drive URL)
│       file_type: TEXT              (계획서|평가서|참고자료|기타)
│
└── activity_suggestions              ← 활동 제안 (미구현)
    └── options: JSONB
        source: TEXT                 (manual | ai)
```

### 9.2 RLS 정책

| 테이블 | 정책 |
|--------|------|
| `profiles` | 모든 인증 사용자 조회 가능, 본인만 수정 |
| `participants` | 당사자는 본인만 조회, 지원자/관리자는 전체 조회 |
| `funding_sources` | `participant_id` 기반 당사자 본인 조회, 지원자/관리자 전체 조회 |
| `transactions` | 당사자 본인 조회 + INSERT(pending), 지원자/관리자 전체 CRUD |
| `plans` | 당사자 본인 조회, 지원자/관리자 전체 CRUD |
| `evaluations` | 당사자 본인 조회, 지원자/관리자 전체 CRUD |
| `file_links` | 당사자 본인 조회, 지원자/관리자 전체 CRUD |

### 9.3 주요 설계 결정

| 결정 | 이유 |
|------|------|
| 재원별 잔액 캐시 (`current_month_balance`) | 잔액 조회 시 매번 transactions 합산 불필요 |
| `transactions.status` (pending/confirmed) 분리 | 당사자 영수증 임시 반영 → 지원자 확정 흐름 |
| `plans.options` JSONB 저장 | 선택지 수가 유동적 (2~3개), 스키마 변경 없이 유연 |
| Google OAuth 도메인 제한 | MVP 단계 보안 강화, 이후 일반 Gmail 허용 검토 |

## 14. 테스트 시나리오

### 시나리오 1: 당사자 등록 및 예산 구조 설정
관리자 로그인 → 새 당사자 등록 → 운영 기간·예산·재원·담당 지원자 설정 → 홈 화면 잔액 표시 확인
> **기대 결과**: 통합 보기와 재원별 보기 모두 가능

### 시나리오 2: 오늘 계획 비교
당사자 홈 → 오늘 계획 보기 → 선택지 카드 2~3개 확인 → 방법 선택 → 미리보기 결과 확인
> **기대 결과**: 선택지별 남는 예산이 다르게 표시됨, 이미지 차이로도 이해 가능

### 시나리오 3: 영수증 업로드 및 임시 반영
당사자 영수증 업로드 → OCR 결과 임시 저장 → 지원자 검토·수정·확정 저장
> **기대 결과**: 당사자 업로드만으로 즉시 확정 차감되지 않음

### 시나리오 4: 달력에서 내역 확인
달력 화면 → 사용일 표시 확인 → 날짜 클릭 → 하루 총 사용액 및 임시/확정 상태 구분 확인
> **기대 결과**: 날짜별 흐름을 쉽게 확인 가능

### 시나리오 5: 월별 평가 및 AI 설명
지원자 4+1 형식 평가 입력 → AI 분석 생성 → 지원자용/당사자용 설명 각각 확인
> **기대 결과**: 같은 평가를 서로 다른 수준의 설명으로 확인 가능

### 시나리오 6: 파일 링크 열람
지원자 Drive 링크 등록 → 당사자 파일 보기 → 링크 목록 확인 → 원본 문서 열기
> **기대 결과**: 허용된 링크만 열람 가능, 자동 공유 없이 링크 기반 연결 작동
