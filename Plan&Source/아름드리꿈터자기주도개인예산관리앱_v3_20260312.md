# 중랑구청 자기주도 개인예산 관리 앱: Antigravity 구현 상세 바이브코딩 계획서

## 문서 정보

- 작성일: 2026년 3월 13일
- 프로젝트명: 중랑구청 자기주도 개인예산 관리 지원 시스템
- 영문명: Areumdree Kkumteo Self-Directed Personal Budget Support System
- 대상 기관: 중랑구청
- 구현 플랫폼: Google Antigravity
- 백엔드 기준: Supabase
- 문서 버전: v3.0 (통합 반영본)

## 1. 프로젝트 배경 및 목적

### 1.1 프로젝트 배경

성인 발달장애인 당사자의 개인예산제 운영에서 가장 중요한 과제는 당사자의 자기결정권을 보장하면서도 실제 생활 속 예산 관리를 이해하기 쉬운 방식으로 지원하는 것입니다.

기존의 일반 예산 관리 도구는 숫자 중심, 표 중심, 관리자 중심으로 설계된 경우가 많아, 의사소통에 어려움이 있거나 추상적인 금액 변화 이해가 어려운 당사자에게는 접근성이 낮습니다. 또한 계획서, 평가서, 영수증, 사용 내역, 파일 첨부 자료가 분산되어 있어 당사자와 지원자가 함께 확인하기 어렵습니다.

중랑구청는 당사자가 스스로 계획하고, 직접 선택하고, 지원자와 함께 확인하는 과정을 통해 자기주도적인 개인예산 관리를 실현하고자 합니다. 이에 따라 본 프로젝트는 예산 변화의 시각화, 선택지 비교, 영수증 OCR, 월별 평가, 파일 링크 공유, AI 분석 보조 기능을 포함한 반응형 웹 앱을 구축하는 것을 목표로 합니다.

### 1.2 프로젝트 목적

본 프로젝트는 다음 목적을 달성하기 위해 추진합니다.

1. 자기주도성 강화: 당사자가 자신의 예산을 스스로 계획하고 확인하는 주체가 되도록 지원
2. 선택의 가시화: 오늘의 선택에 따라 예산과 시간이 어떻게 달라지는지 이미지와 쉬운 문장으로 즉시 이해 가능하도록 지원
3. 지원의 체계화: 지원자가 계획, 기록, 평가를 작성하고 당사자는 이를 쉬운 정보 형식으로 함께 확인할 수 있도록 지원
4. 자료의 통합 관리: 예산 사용, 영수증, 평가, 첨부파일, AI 안내를 당사자별로 체계적으로 관리
5. 현장 활용성 확보: 모바일, 태블릿, PC 환경에서 모두 사용할 수 있는 반응형 웹 시스템 구축

### 1.3 핵심 가치 및 원칙

본 앱은 다음 가치를 기반으로 설계합니다.

- 자기결정 중심: 관리받는 구조가 아니라 스스로 선택하는 구조
- 지원 중심: 서비스 제공보다 선택과 이해를 돕는 지원
- 직관적 시각화: 복잡한 그래프보다 이미지와 크기 변화 중심 표현
- 성인 존중: 유아적인 표현을 지양하고 성인 친화적, 존중하는 문체와 디자인 적용
- 보편적 이해: 말이 적어도, 숫자가 어려워도 이해 가능한 정보 구조
- 현장 적합성: 중랑구청의 개인예산제 운영 구조와 평가 체계 반영

## 2. 기관 운영 구조 및 사용자 정의

### 2.1 기관 운영 구조 반영

중랑구청의 개인예산제는 기본적으로 연 150만원 기준으로 운영합니다.

운영 기간은 2026년 3월부터 2026년 12월까지이며, 월 20만원의 예산 중 15만원을 당사자의 모임통장 체크카드 사용 예산으로 반영하여 입급합니다. 체크카드는 당사자가 직접 보관할 수도 있고, 지원자가 대신 보관할 수도 있으며, 이는 당사자의 선택에 따라 달라집니다.

또한 일부 당사자는 재원이 2개인 경우가 있으므로, 시스템은 다음 두 가지 방식을 모두 지원해야 합니다.

- 통합 보기: 전체 재원을 합산한 잔액 표시
- 재원별 보기: 재원 1, 재원 2를 따로 구분한 잔액 표시

### 2.2 주요 사용자 정의

#### 사용자 1: 당사자

- 성인 발달장애인
- 자신의 예산 현황을 확인함
- 오늘의 선택지에 따른 예산 차이를 비교함
- 영수증 사진을 올림
- 본인 계획 및 평가의 쉬운 설명을 확인함
- 본인에게 연결된 파일 링크를 열람함

#### 사용자 2: 지원자 000

- 당사자별 계획 입력 및 수정
- 예산 정보 등록 및 관리
- 사용 내역 등록, 수정, 삭제, 조회
- 영수증 OCR 결과 검토 및 확정 반영
- PCP 4+1 형식 기반 월별 평가 작성
- AI 분석 결과 확인
- 첨부파일 링크 등록
- 당사자 맞춤 활동 및 선택지 제안 관리

#### 사용자 3: 관리자

- 기관 기본 설정 관리
- 사용자 계정 및 권한 관리
- 당사자 등록 및 예산 구조 설정
- 전체 데이터 운영 점검
- 시스템 공통 문구 및 기준값 관리

### 2.3 사용자 니즈 요약

당사자의 핵심 니즈는 “남은 돈이 얼마나 있는지 눈으로 보고 싶다”, “어떤 방법을 고르면 돈이 얼마나 달라지는지 바로 알고 싶다”, “어려운 설명 없이 쉽게 이해하고 싶다”는 점입니다.

지원자의 핵심 니즈는 “당사자별 자료를 분리해서 안전하게 관리하고 싶다”, “영수증과 기록 입력 시간을 줄이고 싶다”, “평가 내용을 당사자에게 쉽게 설명할 수 있도록 돕고 싶다”는 점입니다.

### 2.4 접근성 고려사항

본 앱은 쉬운 정보 형식을 우선으로 하며 다음 원칙을 적용합니다.

1. 큰 터치 영역 사용
2. 한 화면에 한 가지 핵심 판단 중심 배치
3. 아이콘, 텍스트, 이미지 동시 제공
4. 복잡한 표와 그래프 최소화
5. 삭제 및 변경 전 확인 단계 제공
6. 일관된 버튼 위치와 문구 유지
7. 성인 친화적 색상과 일러스트 사용

## 4. 기능 명세 (Feature Specification)

### 4.1 핵심 기능 1: 당사자별 예산 프로필 관리

**목적**: 당사자마다 개별 예산 구조를 분리 저장하고 운영

**세부 기능**
- 당사자 등록
- 사용 기간 설정
- 기본 월 예산 설정
- 기본 연 예산 설정
- 재원 수 설정
- 재원별 예산 입력
- 통합 예산 자동 계산
- 경고 기준 설정
- 담당 지원자 지정
- 첨부 문서 링크 등록

**주요 데이터 항목**
- participant_id
- name
- monthly_budget_default
- yearly_budget_default
- budget_start_date
- budget_end_date
- funding_source_count
- total_month_balance
- total_year_balance
- alert_threshold
- assigned_supporter
- created_at

### 4.2 핵심 기능 2: 홈 대시보드 및 잔액 시각화

**목적**: 현재 예산 상태를 당사자가 한눈에 이해할 수 있도록 표시

**세부 기능**
- 이번 달 남은 예산 큰 숫자 표시
- 올해 남은 예산 보조 표시
- 통합 보기 / 재원별 보기 전환
- 남은 날짜 표시
- 사용 속도 안내 문구
- 돈주머니/돈뭉치/지갑 이미지 변화
- 오늘 계획 보기 버튼
- 영수증 올리기 버튼
- 이번 달 평가 보기 버튼

**1% 단위 시각화 로직**
- 월 기준 150,000원은 100단계로 구분
- 1단계당 1,500원 반영
- 연 기준 1,500,000원은 100단계로 구분
- 1단계당 15,000원 반영

**상태 문구 예시**
- 81~100%: 예산이 넉넉합니다
- 61~80%: 예산이 안정적입니다
- 41~60%: 예산을 살펴보며 쓰고 있습니다
- 21~40%: 남은 돈이 줄고 있습니다
- 11~20%: 남은 돈이 적습니다
- 0~10%: 모든 예산을 사용했습니다

**남은 날짜 반영 문구 예시**
- 이번 달 남은 날짜에 비해 예산이 여유 있습니다
- 지금 속도로 쓰면 계획에 맞게 사용할 수 있습니다
- 이번 달 남은 날짜를 생각하면 조금 천천히 쓰는 것이 좋습니다

### 4.3 핵심 기능 3: 오늘 계획 비교 (선택 시뮬레이션)

**목적**: 활동 방법에 따른 비용과 시간 차이를 당사자가 직접 비교하고 선택

**세부 기능**
- 활동명 입력 또는 추천 활동 선택
- 방법 2~3개 카드 표시
- 각 방법별 비용 표시
- 각 방법별 소요 시간 표시
- 선택 후 남는 예산 표시
- 선택 후 줄어드는 이미지 비교
- 쉬운 설명 문구 표시
- 미리보기 중심 제공
- 필요 시 지원자가 실제 기록으로 전환 가능

**예시**
활동: 카페 가기

- 방법 A: 택시 / 비용 높음 / 시간 짧음
- 방법 B: 버스 / 비용 낮음 / 시간 김
- 방법 C: 도보 + 버스 / 비용 낮음 / 시간 더 김

각 카드에는 다음 요소를 포함합니다.
- 예상 비용
- 예상 시간
- 선택 후 남는 오늘 예산
- 선택 후 남는 이번 달 예산
- 선택 후 남는 연 예산
- 이미지 차이
- 쉬운 설명 문구

### 4.4 핵심 기능 4: 활동 및 사용 내역 관리

**목적**: 지원자와 관리자가 당사자별 사용 내역을 관리

**세부 기능**
- 등록(Create)
- 조회(Read)
- 수정(Update)
- 삭제(Delete)
- 재원 구분 입력
- 관련 계획 연결
- 관련 평가 메모 연결
- 영수증 첨부
- 입력자 정보 기록
- 수정 이력 기록

**입력 항목**
- 날짜
- 활동 내용
- 사용 금액
- 분류
- 메모
- 결제 수단
- 영수증 이미지
- 입력자
- 재원 구분
- 관련 계획
- 관련 평가 메모

**원칙**
- 관리자와 지원자만 CRUD 가능
- 당사자는 직접 등록 대신 영수증 제출과 확인 참여 중심

### 4.5 핵심 기능 5: 영수증 OCR

**목적**: 영수증 정보를 자동 추출하여 입력 부담 감소

**OCR 모델**
- OpenAI GPT-5 기반 OCR 적용
- 실제 도입 전 비용 검증용 PoC 선행

**기본 흐름**
1. 당사자 또는 지원자가 영수증 사진 업로드
2. OCR이 날짜, 금액, 사용처 후보 추출
3. 결과를 임시 반영 상태로 저장
4. 지원자가 결과 확인
5. 확인 후 실제 예산 사용 내역으로 확정 저장

**화면 요소**
- 영수증 이미지 미리보기
- 자동 읽은 내용 표시
- 날짜 수정
- 금액 수정
- 사용처 수정
- “맞아요”
- “수정할래요”
- “지원자 확인이 필요합니다” 안내

**운영 원칙**
- 당사자 업로드만으로 즉시 차감하지 않음
- OCR 결과는 반드시 검토 단계를 거침
- 오류 수정 가능
- 임시 반영과 확정 반영을 구분 저장

### 4.6 핵심 기능 6: 달력 화면

**목적**: 날짜별 흐름을 중심으로 예산 사용 내역 이해

**세부 기능**
- 월간 달력 표시
- 사용 내역 있는 날 표시
- 날짜 클릭 시 상세 내역 표시
- 하루 총 사용액 표시
- 계획 변경 기록 확인
- 임시/확정 상태 구분 표시
- 재원별 사용 여부 표시 가능

### 4.7 핵심 기능 7: 월별 평가 및 AI 분석

**목적**: 지원자의 평가 기록과 당사자용 쉬운 설명을 연결

**평가 방식**
- 지원자가 직접 평가 작성
- PCP(사람중심생각) 기술 중 4+1 형식 기반 서술형 평가 적용
- 개별 평가도구는 참여자별 별도 붙임 자료로 관리
- 시스템에는 관련 파일 링크 연결

**AI 출력 1: 지원자용 분석**
- 일별 사용 흐름 분석
- 주별 사용 흐름 분석
- 월별 사용 흐름 분석
- 3개월 단위 분석
- 분기별 분석
- 계획 대비 실제 사용 경향 분석
- 반복 선택 패턴 분석
- 위험 구간 및 편차 구간 안내
- 다음 지원 시 참고 포인트 제안

**AI 출력 2: 당사자용 쉬운 설명**
- 이번 기간에 돈을 어떻게 사용했는지 쉬운 설명
- 어떤 선택이 돈을 더 아끼는 데 도움이 되었는지 설명
- 다음 선택 시 참고할 수 있는 짧은 제안
- 어려운 분석 용어를 제외한 간단한 문장 제공

### 4.8 핵심 기능 8: 문서 및 파일 링크 첨부

**목적**: Google Drive 내 자료를 당사자별로 연결하여 공유

### 4.9 핵심 기능 9: 활동 및 선택지 제안

**목적**: 첨부자료와 기존 기록을 바탕으로 새로운 활동 및 선택지 제안

**제안 기준**

- 과거 활동 이력
- 사용 예산 흐름

**예시 제안**

- 시간이 짧지만 비용이 높은 선택지

**운영 원칙**

## 6. UI/UX 상세 설계

### 6.1 디자인 방향

본 시스템은 그래프 중심 화면보다 당사자가 직관적으로 이해할 수 있는 성인 친화적 시각 구조를 우선합니다.

사람과 소통, 한국장애인개발원, 국립특수교육원 등의 쉬운 정보 자료 구성 원칙을 참고하되, 특정 시안을 그대로 복제하지 않고 중랑구청의 운영 흐름에 맞게 재구성합니다.

### 6.2 디자인 시스템

**색상 방향**
- 주요 색상: 차분한 파랑
- 긍정/여유: 초록
- 주의: 주황
- 부족 위험: 빨강
- 카드 배경: 연한 회색
- 본문 배경: 흰색

**타이포그래피**
- 한글 글꼴: 가독성 높은 산세리프 계열
- 숫자: 구분이 쉬운 숫자 전용 서체 병행 가능
- 핵심 숫자: 크게 표시
- 본문 문장: 짧고 읽기 쉬운 크기 유지

**시각 원칙**
- 큰 핵심 숫자
- 쉬운 설명 문장
- 단순한 카드형 배치
- 성인 친화적 일러스트
- 색상만이 아닌 크기와 이미지로 의미 전달
- 버튼 영역 충분히 크게 설계
- 한 화면 한 기능 중심

### 6.3 접근성 원칙

- 텍스트 + 아이콘 + 이미지 동시 제공
- 선택지는 2~3개 중심
- 큰 글씨 모드 지원
- 높은 대비 모드 지원
- 즉시 피드백 제공
- 오류 시 쉬운 문장으로 안내
- 모바일 우선 터치 인터페이스 설계

### 6.4 주요 화면 예시 방향

**홈 화면**
- 당사자 이름
- 이번 달 남은 예산
- 올해 남은 예산
- 통합 보기 / 재원별 보기
- 남은 날짜
- 현재 사용 속도
- 돈주머니/돈뭉치 이미지
- 오늘 계획 보기
- 영수증 올리기
- 이번 달 평가 보기

**오늘 계획 화면**
- 활동명
- 방법 카드 2~3개
- 비용
- 시간
- 선택 후 남는 예산
- 이미지 차이
- 쉬운 설명 문구

**영수증 업로드 화면**
- 사진 촬영 또는 업로드
- 자동 읽은 내용 확인
- 맞아요 / 수정할래요
- 지원자 확인 필요 안내
- 임시 반영 상태 표시

**평가 보기 화면**
- 지원자 작성 평가

- 당사자용 쉬운 설명

## 9. 테스트 시나리오

### 9.1 기능 테스트

#### 시나리오 1: 당사자 등록 및 예산 구조 설정

1. 관리자 로그인
2. 새 당사자 등록
3. 운영 기간 설정
4. 월 예산 및 연 예산 입력
5. 재원 수 설정
6. 담당 지원자 지정
7. 저장
8. 홈 화면에 잔액 정보가 표시되는지 확인

**기대 결과**
- 당사자 정보가 저장됨
- 통합 보기와 재원별 보기 모두 가능함

#### 시나리오 2: 오늘 계획 비교

1. 당사자 홈 화면 진입
2. 오늘 계획 보기 클릭
3. 활동과 선택지 2~3개 확인
4. 각 선택지의 비용, 시간, 남는 예산 이미지 차이 확인
5. 한 방법 선택
6. 미리보기 결과 확인

**기대 결과**
- 선택지별 남는 예산이 다르게 표시됨
- 이미지 차이로도 결과를 이해할 수 있음

#### 시나리오 3: 영수증 업로드 및 임시 반영

1. 당사자가 영수증 업로드
2. OCR 결과 자동 추출
3. 임시 반영 상태 저장
4. 지원자가 검토 화면 진입
5. 수정 또는 확인
6. 확정 저장

**기대 결과**
- 당사자 업로드만으로 즉시 확정 차감되지 않음
- 지원자 확인 후 최종 반영됨

#### 시나리오 4: 달력에서 내역 확인

1. 달력 화면 진입
2. 사용일 표시 확인
3. 특정 날짜 클릭
4. 하루 총 사용액 확인
5. 임시/확정 상태 구분 확인

**기대 결과**
- 날짜별 흐름을 쉽게 확인 가능함

#### 시나리오 5: 월별 평가 및 AI 설명 확인

1. 지원자가 4+1 형식 평가 입력
2. AI 분석 생성
3. 지원자용 분석 확인
4. 당사자용 쉬운 설명 확인

**기대 결과**
- 같은 평가를 서로 다른 수준의 설명으로 확인 가능함

#### 시나리오 6: 파일 링크 열람

1. 지원자가 Google Drive 링크 등록
2. 당사자 계정으로 파일 보기 화면 진입
3. 링크 목록 확인
4. 원본 문서 열기

**기대 결과**
- 허용된 링크만 열람 가능함
- 자동 공유 없이 링크 기반 연결이 작동함

### 9.2 접근성 테스트

#### 시나리오 7: 큰 글씨 모드

- 모든 주요 텍스트 확대
- 버튼과 레이아웃 유지 확인

#### 시나리오 8: 이미지 중심 이해 가능성

- 숫자 읽기 없이도 잔액 이미지로 상태 파악 가능 여부 확인
- 선택지 카드 간 이미지 차이만으로 비교 가능 여부 확인

#### 시나리오 9: 반응형 검증

- 모바일, 태블릿, PC에서 레이아웃과 기능이 모두 정상 동작하는지 확인
